Starbucks Offer Analysis

Overview

This project explores simulated Starbucks transaction data to predict how customers respond to different offer types. The goal is to build a model that can identify the most effective offers for each customer, improving the return on marketing spend and enhancing customer experience.

Objective

The main objective is to use machine learning to predict whether a customer will respond positively to an offer. This involves:

  • Classifying offers as successful or unsuccessful based on customer behavior.
  • Building a predictive model to identify the best offer for each customer.
  • Comparing the performance of different machine learning algorithms, including a neural network.

Process

1. Data Gathering

The project uses three simulated datasets provided by Udacity:

  • portfolio.json: Contains information about the different offers, such as reward, difficulty, duration, and offer type.
  • profile.json: Contains demographic data for each customer, including gender, age, income, and membership date.
  • transcript.json: Contains records of transactions, offers received, offers viewed, and offers completed.

2. Data Exploration and Preparation

The datasets were explored to understand the data distribution, identify patterns, and clean any inconsistencies. This involved:

  • Analyzing the distribution of offer characteristics.
  • Identifying and handling missing values in customer demographics.
  • Visualizing the frequency of different events over time.
  • Analyzing the relationship between offer characteristics and customer response.

3. Feature Engineering

New features were engineered to improve the model's predictive power. These included:

  • Total spending by each customer prior to receiving an offer.
  • Total number of transactions made by each customer.
  • Number of previous offers received, viewed, and completed by each customer.

4. Model Building

Two types of models were built and evaluated:

  • Benchmark Model: A Random Forest Classifier from the sklearn library was chosen as the benchmark model. GridSearchCV was used to optimize the model's hyperparameters for maximum recall.
  • Neural Network Model: A linear neural network was built using PyTorch. The network architecture, loss function, and optimizer were tuned to achieve optimal performance.

Results

Both models were evaluated on a held-out test set (15% of the data). The neural network model slightly outperformed the benchmark model in terms of recall, accuracy, and F1-score.

Conclusion

This project demonstrated the effectiveness of machine learning in predicting customer response to marketing offers. The neural network model showed promising results, suggesting that deep learning techniques can be valuable in this domain.

Future Improvements

Potential areas for improvement include:

  • Exploring more complex neural network architectures.
  • Incorporating additional features, such as time lag between offer receipt and view.
  • Introducing a new class for "very unsuccessful" offers where customers complete an offer without viewing it.
